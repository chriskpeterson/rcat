Information Architecture

Outcome-Oriented Strategy

Modern documentation must serve two masters: the developer who needs to ship code now, and the stakeholder who needs to understand the why.

The Model

Dual-Track Content: Task vs. Concept

Documentation often fails because it mixes theory with execution. I propose a hard separation that allows users to self-select their depth.

Track A: The Task (Do)

Pure execution. Code snippets, CLI commands, and dashboard steps. No fluff. Perfect for "vibe coders" and rapid integration.

Track B: The Concept (Understand)

The mental model. Why entitlements matter, how offerings scale. Essential for architects or as context for RAG/AI agents.

Operational Strategy

The Single Source of Truth

Technical knowledge often hides in repositories, Jira, and tribal memory. I architect documentation as the final, governed "Single Source of Truth." While I advocate for open contribution from anyone in the organization, the documentation team serves as the essential gatekeeper—editing, refining, and ensuring content is ready for both human developers and AI agents.

Gap 01: Foundational Friction

The "Dashboard-First" Prerequisite

Current docs force developers through extensive manual configuration before writing a line of code. This creates a high barrier to entry for early experimentation.

Fix: Reposition the Test Store as the primary quickstart path. Enable code execution in 60 seconds, and defer dashboard complexity until the user has seen the "magic" of a successful purchase flow.

Gap 02: Labeling & Structure

The Entitlement Fog

Developers consistently confuse Store Products with RevenueCat Entitlements. The current documentation treats these as isolated topics rather than a unified flow.

Domain Primary Goal
Product Catalog Source of Truth configuration.
Implementation Pure task-based coding & lifecycle.
Growth Optimization via remote config.
Operations Data flow, webhooks, & analytics.

AI Strategy

Structured for Machine-Reading

By strictly separating tasks from concepts, the documentation becomes chunk-friendly for AI. AI agents can easily retrieve "The Task" for a developer asking for code, while using "The Concept" to provide high-level reasoning without cluttering the IDE snippet.
