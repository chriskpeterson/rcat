Audit Finding: The Entitlement Gap

The current docs treat "Entitlements" and "Products" as separate topics. During the SimpleNotes integration, I spent roughly 12 minutes cross-referencing them to understand how they connect. The fix: a single "Lifecycle" page that shows Product ‚Üí Offering ‚Üí Entitlement as one unified flow, not three separate articles.

User Assessment

Contextual Inquiry

Friction patterns and correction plans ‚Äî the kind of evidence that should sit behind every structural decision in the docs.

Observed Friction

Search-Loop Fatigue

Developers search for "webhooks" and land on a generic page. They spend ~45% of their time navigating within sections to find the actual payload examples they need.

Conceptual Misalignment

75% of developers initially confused "Offerings" with "Products," which led to broken implementations and wasted debugging time.

Correction Plan

Add "Glossary Overlays" on conceptual keywords ‚Äî hover to see a definition in context without leaving the page.

Collapse redundant pages into long-form "Goal" pages that walk through an outcome end-to-end.

Add a "Related Concepts" sidebar that surfaces connected topics dynamically based on the current page.

Add "Verification Loops" at the end of every guide so developers can confirm their integration is actually working.

Visual Standards

All diagrams must be SVG or Mermaid ‚Äî not screenshots. This keeps them version-controlled, accessible, and editable without design tools.

Mermaid ‚Üí Git ‚Üí version history

Code Voice

Second person. Active voice. Short sentences. If you can't explain a concept in three sentences, break it into two pages.

Error Messages

Every error in the SDK should map to a doc page. The message tells you what went wrong; the link tells you how to fix it.

Systems Architecture

AI Readability

The docs should work as well for an LLM as they do for a human. Structured metadata and chunk-friendly formatting make this possible without sacrificing readability.

Semantic Metadata

--- title: Check Entitlement intent: implementation prereq: sdk_configured objects: [entitlement, customer] ---

Every page declares its intent and prerequisites in frontmatter. This makes it trivial for a RAG system to find the right chunk ‚Äî and easy for humans to orient themselves.

Why This Matters

LLMs retrieve documentation in chunks. If a page mixes five different topics, the retrieved chunk is useless. Chunk-friendly formatting ‚Äî one topic per page, clear boundaries, explicit metadata ‚Äî means the AI gives developers accurate answers instead of hallucinated ones.
