Configuration

Store Connection

90% of "why isn't this working?" issues come from Apple or Google blocking the connection. Check every item here before you debug any code.

Pre-Flight Checklist

Paid Apps Agreement Signed

Go to App Store Connect > Agreements. If this isn't active, Apple rejects all payments ‚Äî no exceptions.

Critical

Product Status is "Ready to Submit"

In App Store Connect > Subscriptions. If it says "Missing Metadata," you need to add a screenshot and description before anything will work.

Critical

Shared Secret Copied

Found in App Store Connect > Users and Access > Shared Secret. Paste this into your RevenueCat project settings.

Bundle ID Matches Exactly

Your RevenueCat app's Bundle ID must be character-for-character identical to the one in your Xcode project. A single typo breaks everything.

Foundation

Core Concepts

RevenueCat renames some platform terms to make them work cross-platform. Here's the translation ‚Äî and how the pieces connect.

Entitlements

"What the user gets."

For example, "Pro Access." This is the only thing you check in your code to verify whether a user has an active subscription. Everything else is setup.

Offerings

"The Menu."

The collection of products you're showing on your paywall right now. You can change which Offering is "current" in the RevenueCat dashboard without touching your code.

Packages

"The Buttons."

A specific product ‚Äî like "Annual Plan" ‚Äî attached to an Offering. Each Package maps to one product on each platform (iOS, Android, Web).

Products

"The Store Listings."

The actual subscription you created in App Store Connect or Google Play. RevenueCat doesn't create these ‚Äî it just references them.

Using a visual builder? You don't need the SDK code from the walkthrough ‚Äî but you do need to configure the connection manually.

FlutterFlow

FlutterFlow has a native RevenueCat integration. You do not need custom actions for basic subscriptions.

1

Go to Settings > RevenueCat. Toggle "Enable RevenueCat."

2

Paste your Apple and Google API Keys from the RevenueCat dashboard.

3

Click "Deploy to App Store" at least once. Test Mode won't work until a binary has been uploaded.

Warning

The "Test" button in FlutterFlow often fails for payments. Download the APK or IPA and test on a real physical device.

Troubleshooting

Error Code Reference

When something breaks, the SDK returns an error code. Here's what each one actually means ‚Äî in plain language.

23 ‚Äî ConfigurationError Most Common

Empty Offerings / Invalid Config

RevenueCat asked the App Store for your products, and the App Store said "I don't know those." This is almost always a setup problem, not a code problem.

How to fix it:
  • Make sure the "Paid Applications Agreement" is signed in App Store Connect.
  • Make sure your products are "Ready to Submit" ‚Äî not "Missing Metadata."
  • Double-check that your Bundle ID matches exactly between Xcode and RevenueCat.
1 ‚Äî PurchaseCancelled Normal Behavior

User Cancelled

The user saw the price and tapped the X button or swiped the sheet away. This is not an error ‚Äî it's expected.

How to handle it:

Do not show an error message to the user. Check the userCancelled boolean and fail silently.

8 ‚Äî InvalidCredentials

Wrong API Key

Your code is using the wrong key for RevenueCat. This is usually a copy-paste mix-up between public and secret keys.

How to fix it:

Check your .configure() call. You should be using a Public key ‚Äî starts with appl_ or goog_. If it starts with sk_, that's the secret key ‚Äî it should never be in your app.
